## Overview

Memento AI supports multiple distribution channels with automatic updates via Velopack.

<Info>
  **Primary Method**: GitHub Releases  
  **Alternative**: Self-hosted CDN  
  **Update Check**: Every 6 hours
</Info>

## GitHub Releases

### Setup

<Steps>
  <Step title="Create release">
    ```bash
    # Tag version
    git tag v1.0.0
    git push origin v1.0.0
    
    # Create release
    gh release create v1.0.0 \
      --title "Memento AI v1.0.0" \
      --notes "Release notes here"
    ```
  </Step>
  
  <Step title="Upload assets">
    ```bash
    gh release upload v1.0.0 \
      ./velopack-output/Setup.exe \
      ./velopack-output/RELEASES \
      ./velopack-output/memento-1.0.0-full.nupkg \
      ./velopack-output/memento-1.0.0-delta.nupkg
    ```
  </Step>
  
  <Step title="Configure Velopack">
    ```rust
    let manager = UpdateManager::new(UpdateOptions {
        url: "https://github.com/your-org/memento-ai/releases".to_string(),
        ..Default::default()
    })?;
    ```
  </Step>
</Steps>

### Release Manifest

Velopack requires a `RELEASES` file:

```
SHA256_HASH  memento-1.0.0-full.nupkg  85234567
SHA256_HASH  memento-1.0.0-delta.nupkg  3456789
```

This is automatically generated by `vpk pack`.

## Self-Hosted CDN

### Directory Structure

```
cdn.yourcompany.com/
└── releases/
    ├── RELEASES                      # Manifest
    ├── memento-1.0.0-full.nupkg
    ├── memento-1.0.0-delta.nupkg
    ├── memento-0.9.0-full.nupkg
    └── memento-0.9.0-delta.nupkg
```

### Nginx Configuration

```nginx
server {
    listen 443 ssl;
    server_name cdn.yourcompany.com;
    
    ssl_certificate /etc/ssl/cert.pem;
    ssl_certificate_key /etc/ssl/key.pem;
    
    root /var/www/releases;
    
    location /releases/ {
        autoindex on;
        add_header Cache-Control "public, max-age=3600";
        
        # CORS for update checks
        add_header Access-Control-Allow-Origin "*";
    }
    
    # Analytics
    access_log /var/log/nginx/cdn-access.log combined;
}
```

### Update Manager Config

```rust
let manager = UpdateManager::new(UpdateOptions {
    url: "https://cdn.yourcompany.com/releases".to_string(),
    ..Default::default()
})?;
```

## Microsoft Store

### Preparation

<Steps>
  <Step title="Create developer account">
    Register at [Partner Center](https://partner.microsoft.com/).
  </Step>
  
  <Step title="Configure app identity">
    ```xml
    <!-- Package.appxmanifest -->
    <Identity
      Name="YourCompany.MementoAI"
      Publisher="CN=Your Company"
      Version="1.0.0.0" />
    ```
  </Step>
  
  <Step title="Build MSIX package">
    ```powershell
    # Install Windows SDK
    winget install Microsoft.WindowsSDK
    
    # Build MSIX
    makeappx pack /d .\release-staging /p memento.msix
    
    # Sign package
    signtool sign /fd SHA256 /f cert.pfx /p password memento.msix
    ```
  </Step>
  
  <Step title="Submit to Store">
    Upload MSIX to Partner Center with screenshots, description, and privacy policy.
  </Step>
</Steps>

## Installation Methods

### Silent Install

For enterprise deployments:

```powershell
# Silent install (no UI)
.\Setup.exe --silent

# Install to custom directory
.\Setup.exe --installPath "C:\Program Files\Memento"

# Skip desktop shortcut
.\Setup.exe --noShortcut
```

### MSI Installer

Convert Velopack package to MSI:

```powershell
# Use WiX Toolset
heat dir .\release-staging -out files.wxs
candle files.wxs
light -out memento.msi files.wixobj
```

## Auto-Update Configuration

### Update Channels

Support multiple release channels:

```rust
let channel = match env::var("RELEASE_CHANNEL") {
    Ok(ch) => ch,
    Err(_) => "stable".to_string()
};

let url = format!(
    "https://github.com/your-org/memento-ai/releases/channel/{}",
    channel
);

let manager = UpdateManager::new(UpdateOptions {
    url,
    ..Default::default()
})?;
```

**Channels**:
- `stable`: Production releases
- `beta`: Pre-release features
- `nightly`: Daily builds

### Custom Update Frequency

```rust
let manager = UpdateManager::new(UpdateOptions {
    url: "...".to_string(),
    check_interval_hours: 12,  // Default: 6
    ..Default::default()
})?;
```

## Analytics

### Download Tracking

Track downloads with GitHub API:

```bash
# Get download count
gh api repos/your-org/memento-ai/releases/latest \
  --jq '.assets[] | "\(.name): \(.download_count)"'
```

### Custom Analytics

Add query parameters to update URLs:

```rust
let manager = UpdateManager::new(UpdateOptions {
    url: format!(
        "https://cdn.yourcompany.com/releases?user={}&version={}",
        user_id, current_version
    ),
    ..Default::default()
})?;
```

Server logs:

```
GET /releases?user=abc123&version=1.0.0 200
GET /releases?user=def456&version=0.9.0 200
```

## Enterprise Deployment

### Group Policy

Distribute via Active Directory:

```powershell
# Create GPO
New-GPO -Name "Memento AI Installation"

# Link to OU
New-GPLink -Name "Memento AI Installation" -Target "OU=Workstations,DC=company,DC=com"

# Add software installation
Set-GPRegistryValue -Name "Memento AI Installation" `
  -Key "HKLM\Software\Policies\Memento" `
  -ValueName "InstallPath" `
  -Type String `
  -Value "\\fileserver\software\memento\Setup.exe"
```

### SCCM Deployment

```xml
<!-- Deploy.xml -->
<Application>
  <Name>Memento AI</Name>
  <Version>1.0.0</Version>
  <InstallCommand>Setup.exe --silent</InstallCommand>
  <UninstallCommand>Update.exe --uninstall</UninstallCommand>
  <DetectionMethod>
    <RegistryKey>HKLM\Software\Memento\Version</RegistryKey>
    <Value>1.0.0</Value>
  </DetectionMethod>
</Application>
```

## Versioning Strategy

### Semantic Versioning

Follow [semver](https://semver.org/):

- **Major** (1.0.0): Breaking changes
- **Minor** (0.1.0): New features, backward compatible
- **Patch** (0.0.1): Bug fixes

### Version Tagging

```bash
# Create annotated tag
git tag -a v1.0.0 -m "Release 1.0.0: Initial stable release"

# Push tags
git push origin v1.0.0

# List tags
git tag -l "v*"
```

## Rollback Strategy

If a release has critical bugs:

<Steps>
  <Step title="Identify previous stable version">
    ```bash
    gh release list
    ```
  </Step>
  
  <Step title="Mark new release as pre-release">
    ```bash
    gh release edit v1.0.0 --prerelease
    ```
  </Step>
  
  <Step title="Notify users">
    Send in-app notification to downgrade:
    ```rust
    manager.apply_updates_and_restart(previous_version)?;
    ```
  </Step>
</Steps>

## Best Practices

<Card title="Test Updates Before Release" icon="flask">
  Always test update flow locally:
  
  ```powershell
  # Install old version
  .\memento-0.9.0-Setup.exe
  
  # Build new version
  .\scripts\build-release.ps1 -Version "1.0.0"
  ```

## Troubleshooting

<AccordionGroup>
  <Accordion title="Update not detected" icon="magnifying-glass">
    **Check**:
    1. RELEASES file exists on server
    2. Version number higher than current
    3. Network connectivity
    
    **Debug**:
    ```rust
    let info = manager.check_for_updates().await?;
    println!("Available version: {:?}", info);
    ```
  </Accordion>
  
  <Accordion title="Download fails" icon="download">
    **Common causes**:
    - CDN rate limiting
    - Incomplete upload
    - Network timeout
    
    **Solution**: Retry with exponential backoff
    ```rust
    for attempt in 1..=3 {
        match manager.download_updates(info).await {
            Ok(_) => break,
            Err(e) if attempt < 3 => {
                sleep(Duration::from_secs(2u64.pow(attempt))).await;
            }
            Err(e) => return Err(e),
        }
    }
    ```
  </Accordion>
</AccordionGroup>
