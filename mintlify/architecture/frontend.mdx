## Overview

The user-facing layer is a **Next.js 16** application running inside a **Tauri 2** desktop shell. Users can also run it as a standalone web app at `http://localhost:3000`.

---

## Stack

| Technology | Version | Role |
|---|---|---|
| Next.js | 16 | React framework, routing, SSR |
| React | 19 | UI rendering |
| Tailwind CSS | 3 | Utility-first styling |
| Radix UI | latest | Accessible component primitives |

## Running the frontend

```bash
cd app/frontend
npm run dev     # development (http://localhost:3000)
npm run build   # production build
npm start       # serve production build
```

## Tauri desktop app

The Tauri shell (`app/src-tauri/`) wraps the Next.js frontend into a native desktop window.

```bash
# Dev mode (hot-reload)
cd app && npm run tauri dev

# Production build
cd app && npm run tauri build
```

Built installers land in `app/src-tauri/target/release/bundle/`:

| Platform | Installer format |
|---|---|
| Windows | `.msi`, `.exe` (NSIS) |
| macOS | `.dmg`, `.app` |
| Linux | `.AppImage`, `.deb` |

### Tauri capabilities

Tauri capabilities (fine-grained OS permission grants) are defined in `app/src-tauri/capabilities/`. They control which Tauri APIs the frontend JS can call — e.g., `fs:read`, `dialog:open`.

## Agents (`app/agents/`)

LangChain + LangGraph agents run as a Node.js process and handle multi-step query planning:

1. **Query rewriting** — normalise and expand user queries
2. **Tool selection** — decide between local search, file lookup, or LLM answer
3. **Result synthesis** — merge multiple tool outputs into a ranked, cited response

```bash
cd app/agents
npm run dev
```

Agents communicate with the AI Gateway over HTTP and with the Rust daemon via the local REST API on `localhost:5100`.
